## Catálogo de rutas del sistema PRONTO

### Objetivo
Este catálogo complementa `SYSTEM_ROUTES_SPEC.md` y `SYSTEM_ROUTES_MATRIX.md`. Aquí se documentan **todas las rutas del sistema agrupadas por familia/patrón**, indicando el servicio donde viven, su módulo Python origen y su propósito operativo.

### Fuentes de verdad
- `url_map` real de `pronto-api` con `PRONTO_ROUTES_ONLY=1`
- `url_map` real de `pronto-client` con `PRONTO_ROUTES_ONLY=1`
- `url_map` real de `pronto-employees` con `PRONTO_ROUTES_ONLY=1`
- `pronto-docs/contracts/pronto-api/openapi.yaml`

### Cómo leer este catálogo
- **Familia / patrones**: agrupa todas las rutas registradas hoy bajo un mismo módulo funcional.
- **Métodos**: métodos HTTP observados en el `url_map` actual.
- **Módulo origen**: módulo Flask/Python donde vive la implementación.
- **Naturaleza**: `web`, `api`, `bff`, `proxy`, `compat`.
- **Auth/CSRF**: interpretación operativa resumida; el enforcement exacto sigue en código.

## 4) Observaciones importantes

- El `url_map` actual todavía expone **aliases/compat** en varias familias (`/api/login`, `/api/logout`, `/api/config*`, `/api/menu`, `/api/shortcuts`, `/api/tables*`, `/api/areas*`).
- `pronto-client` y `pronto-employees` siguen teniendo rutas `bff`/`proxy` de transporte; la autoridad de negocio sigue siendo `pronto-api:/api/*`.
- Para ver request/response exactos, usa `pronto-docs/contracts/pronto-api/openapi.yaml`.
- Para ejemplos `curl`, usa `pronto-docs/pronto-api/API_CONSUMPTION_EXAMPLES.md`.
