# Customer App Prompts (Chunk 1)

This file contains a set of prompts for generating code or test cases for the customer-facing application, based on the extracted rules and workflows.

## Authentication

*   **User Registration:** "As a new user, I want to be able to register for an account with my email and password so that I can place orders."
*   **User Login:** "As an existing user, I want to be able to log in to my account using my email and password to access my order history and place new orders."
*   **User Logout:** "As a logged-in user, I want to be able to log out of my account to securely end my session."

## Ordering

*   **Browse Menu:** "As a customer, I want to be able to browse the menu, view item details (including price and description), and see which items are available."
*   **Add to Cart:** "As a customer, I want to be able to add items to my cart, specifying the quantity and selecting any required modifiers."
*   **View Cart:** "As a customer, I want to be able to view the contents of my cart at any time and see the subtotal."
*   **Checkout Process:** "As a customer, I want to be able to proceed to checkout, where I must select my table before I can confirm my order."
*   **Order Status:** "As a customer, I want to be able to see the real-time status of my active orders (e.g., Preparing, Ready, Delivered)."

## Payment

*   **Request Bill:** "As a customer, I want to be able to request my bill when I am ready to pay."
*   **Stripe Payment:** "As a customer, I want the option to pay for my order using a credit or debit card via Stripe."
*   **Split Bill:** "As a customer, I want to be able to split the bill with others at my table."

## Feedback

*   **Provide Feedback:** "As a customer, I want to be able to provide feedback and a rating on my dining experience after my order is complete."

## Other

*   **Call Waiter:** "As a customer, I want a button to call a waiter to my table for assistance."
